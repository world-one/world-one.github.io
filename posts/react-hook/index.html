React Hook

리액트 훅(React Hook)

훅은 함수형 컴포넌트에 기능을 추가할 때 사용하는 함수다. 함수형 컴포넌트에서 상태값을 사용 할 수 있다. 훅이 나오면서 클래스보다는 함수형 컴포넌트로 작성하는 걸 추천한다.

훅 종류

  • useState
  • useEffec
  • useContext
  • useRef
  • useMemo, useCallback
  • useReducer
  • useImperativeHandle
  • useLayoutEffec, useDebugValue

useState

상태값 변경 함수
상태값이 변경 되면 리액트는 해당 컴포넌트를 다시 그린다.
자식 컴포넌트까지 다시 렌더링 된다.
비동기로 동작, 상태값 변경 요청은 배치로 처리(일괄처리)
호출 순서가 보장된다. 비동기로 처리되지만 순서대로??

useEffect

부수효과는 함수 실행시 함수 외부의 상태를 변경하는 연산
렌더링 결과가 실제 돔에 반영된 후 호출, (비동기로 호출) 컴포넌트가 사라지기 직전에 마지막으로 호출..
의존성 배열

useEffect(
  () => {
    getUserApi(userId).then(data => setUser(data));
  },
  [userId],
);

렌더링시마다 호출되는건 비효율적이기 때문에 두번째 매개변수로 의존성 배열을 추가하면 된다.
의존성 배열의 값이 변할 경우만 호출된다. 

return () => {
  window.removeEventListener('resize', onResize);
};
}, []);

반환된 함수는 useEffect(부수효과함수)가 호출되기 직전에 호출되고, 컴포넌트가 사라지기 직전에 호출
의존성 배열을 빈 배열로 넣을 경우 컴포넌트가 생성될 때만 부수 효과 함수가 호출되고, 컴포넌트가 사라질 때 반환된 함수가 호출

Custom Hook

function useUser(userId) {
  const [user, setUser] = useState(null);
  useEffect(() => {
    getUserApi(userId).then(data => setUser(data));
  }, [userId]);
  return user;
}

function Profile({ userId }) {
  const user = useUser(userId);
}

내부 구현을 숨기면서 사용 편의성을 높인다.

Hook 사용시 규칙

  • 하나의 컴포넌트에서 훅을 호출하는 순서는 항상 같아야 한다.
  • 함수형 컴포넌트 또는 커스텀 훅 안에서만 호출되어야 한다.

리액트는 훅이 사용된 순서를 저장하고 그것을 기반으로 훅을 관리한다.

useContext

context api의 consumer를 이용하지 않고 부모 컴포넌트에서 전달된 컨텍스트 데이터를 사용할 수 있다.

const UserContext = React.createContext();
const user = { name: 'mike', age: 23 };

function ParentComponent() {
  return (
    <UserContext.Provider value={user}>
      <ChildComponent />
    </UserContext.Provider>
  )
}

function ChildComponent() {
  const user = useContext(UserContext);
  console.log(`user: ${user.name}, ${user.age}`);
}

useRef

import React, { useRef, useEffect } from 'react';

fucntion TextInput({ inputRef }) {
  return (
    <div>
      <input type="text" ref={inputRef} />
      <button>저장</button>
    </div>
  )
}

function Form(){
  const inputRef = useRef();
  useEffect(() => {
    inputRef.current.focus();
  }, []);
  return (
    <div>
      <TextInput inputRef={inputRef} />
      <button onClick={() => inputRef.current.focus()}>텍스트로 이동</button>
    </div>
  );
}

forwardRef를 이용하면 직접 ref 속성값을 처리할 수 있다.

자식 요소에 접근외에도 중요한 용도로, 컴포넌트 내부에서 렌더링과 무관한 값을 저장할 때 사용된다. 

const prevAgeRef = useRef(20);
const prevAge = prevAgeRef.current;

useMemo, useCallback

이전 값을 기억해서 성능을 최적화하는 용도

  • useMemo
    계산량이 많은 함수의 반환값을 재활용

    import { runExpensiveJob } from './util';
function  MyComponent({ v1, v2 }) {
const value = useMemo(() => runExpensiveJob(v1, v2), [v1, v2]);
return <p>{`value is ${value}`</p>
}

    의존성 배열이 변경되지 않으면 기존에 계산된 값을 재사용한다.

  • useCallback

리액트의 렌더링 성능을 위해 제공되는 훅
속성값이 변할 경우 React.memo를 사용해도 불필요한 렌더링이 발생한다. 

import { saveToServer } from './api';
import UserEdit from './UserEdit';
function Profile() {
  const [name, setName] = useState('');
  const [age, setAge] = useState(0);
  return(
    <div>
      <p>{`name is ${name}`}</p>
      <UserEdit 
        onSave={() => saveToServer(name, age)}
        setName={setName}
        setAge={setAge}
      />
    </div>
  );
}

렌더링될 때마다 onSave에 새로운 함수가 입력된다.

const onSave = useCallback(() => saveToServer(name, age), [name, age]);

의존성 배열이 변경되지 않으면 이전에 생성한 함수가 재사용된다.

useReducer

리덕스의 리듀서처럼 관리할 수 있다. 

import React, { useReducer } from 'react';

const INITIAL_STATE = { name: 'empty', age: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'setName': return { ...state, name: action.name };
    case 'setAge': return { ...state, age: action.age };
    default: retrun state;
  }
}

function Profile() {
  const [state, dispatch] = useReducer(reducer, INITIAL_STATE);
  return (
    <div>
      <p>{`name is ${name}`}</p>
      <input 
        type="text"
        value={state.name}
        onChange={e => dispatch({ type: 'setName', name: e.currentTarget.value })}
      />
    </div>
  );
}

리덕스의 리듀스 방식으로 작성된 함수

컨텍스트 API와 함께 이벤트 처리 함수 전달

export const ProfileDispatch = REact.createContext(null);


function Profile() {
  const [state, dispatch] = useReducer(reducer, INITIAL_STATE);
  return (
    <p>{`name is ${name}`}</p>
    <ProfileDispatch.Provider value={dispatch}>
      <SomeComponent />
    </ProfileDispatch>
  );
}

useReducer 훅의 dispatch 함수는 값이 변하지 않는 특징이 있어 불필요하게 렌더링 되는 일은 발생하지 않는다.

useImperativeHandle

부모 컴퍼넌트에서 접근 가능

useImperativeHandle(ref, () =>({
  addAge: value => setAge(age + value),
  getNameLength: () => name.length,
}))

export default forwardRef(Profile);

useLayoutEffec, useDebugValue

  • useLayoutEffec
    useEffec와 비슷하지만 동기로 호출한다.
    성능상 useEffect를 사용,
    렌더링 직후 돔 요소의 값을 읽는 경우에는 useLayoutEffect 훅을 사용하는 것이 적합

  • useDebugValue
    개발 편의를 위해 제공
    커스텀 훅의 내부 상태를 관찰할 수 있다. 

    function useToggle(initialValue) {
const [value, setValue] = useState(initialValue);
const onToggle = () => setValue(!value);
useDebugValue(value ? 'on' : 'off');
return [value, onToggle];
}
