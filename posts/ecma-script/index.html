ECMA Script

ES?? ECMA??

ECMA Script를 줄여서 ES라고 한다.
자바스크립트와 같은 스크립터 언어를 표준화 하기 위한 스크립트 프로그래밍 언어이며,
Ecma International(정보/통신시스템을 위한 국제적 표준화 기구)에서 ECMA-262 기술 규격에 정의되어 있다.
물론 이를 구현한 언어중 자바스크립트가 가장 유명하고 대표적이라 아예 같다고 생각하는 것도 무리는 아니다.
아무래도 각 브라우저마다 지원하는 기능이 다르면 여러모로 힘들다..
이런 기구가 있어서 매우 감사합니다.

ES6, ES2015?

처음에는 다른건가 싶었다.
ECMA-262는 표준에 의한 작업이 계속 진행되고 있으며, 출판될 때마다 해당판의 숫자도 올라가게 된다. ES6는 그냥 여섯번째 판이라는 뜻이다.
ES6 === ES2015 === ECMAScript2015
버전별 숫자를 붙이는 것 보다는 연도를 붙이는게 맞나보다.

각 버전의 특징?

최근에 나온 것들 위주로 살펴보겠다.

ES5

  • 배열 관련 메소드

    • forEach
    • map
    • filter
    • reduce
    • some
    • every

  • 객체 관련 메소드

    • Object.create()
    • Object.defineProperty()
    • Object.freeze()
    • Object.assign()
    • getter, setter
    • Object.keys()
  • use strict
'use strict';
or
function name(){
  'use strict';
}
  • bind()
const module = {
  x: 42,
  getX: function() {
    return this.x;
  }
};

const unboundGetX = module.getX;
console.log(unboundGetX()); // The function gets invoked at the global scope
// expected output: undefined

const boundGetX = unboundGetX.bind(module);
console.log(boundGetX());
// expected output: 42

  • JSON( Javascript Object Notation )

    • JSON.stringify(obj);
    • JSON.parse(json);
  • LexicalEnvironment
    현재 실행 환경에 대한 변수나 참조에 대한 정보를 가진다.
    https://medium.com/@kkak10/lexical-environment-4e0cffcad98d https://meetup.toast.com/posts/129
    복잡하다.. 다시 조사해서 별도로 작성해보자.

ES2015

  • Class
  • Promise
  • const, let
  • arrow function : () => {}
  • rest parameter
  • spread 연산자
  • data: data => data : 변수와 속성명이 같을 경우 하나만 써도 된다.
  • 동적 속성 생성 : object[ a+3 ], object[ “na”+b ]
  • 템플릿 문자열 : ${var}, wow 백틱을 사용하여 문자열 만들기

ES2016

  • 제곱연산자 : Math.pow(3, 3) == 3 ** 3, i **= 3
  • array.includes( valueToFind[, fromIndex] ) : return boolean
    indexOf와 비슷하지만, NaN 처리에 있어 차이가 있다. includes 찾고 indexOf는 못찾는다.

ES2017

  • Object.values(obj) : value로 배열을 생성해준다.
  • Object.entries(obj) : key, value를 2차원 배열로 만든다.
  • Object.getOwnPropertyDescriptors(obj) : 상속받지 않은 속성들의 설명을 보여준다.
  • String.padStart(targetLength [, padString]) : 앞부분에 공백 또는 문자열 추가
  • String.padEnd(targetLength [, padString]) : 뒷부분에 공백 또는 문자열 추가

  • Trailing commas (final commas ) : 함수의 인자나 매개변수 마지막에 콤마 붙이기 가능

    • 목적 : 코드편집, 버전관리의 용이성
    • 초기 : 배열 리터럴에 허용
    • ES5 : 객체 리터럴
    • ES2017 : 함수의 매개변수
  • async/await

ES 2018

  • Object rest
  const { a, ...rest } = { a: 1, b: 2, c: 3 };
  console.log(rest); // { b: 2, c: 3}
  const { d, g, ...rest2 } = { d: 4, e: { f: 6 }, g: 7, h: 8 }
  console.log(rest2); // { e: { f: 6 }, h: 8 } ( { f: 6 }은 복사가 아니라 참조)
