리액트로 다시 만들어보자

신입시절 부족하지만 그나마 취업에 도움을 주었던 포트폴리오 사이트를 오랜만에 꺼내보았다.
html tag는 그때 당시 웹표준에 맞춰 잘 배웠던 터라 지금과 비교해도 큰 차이는 없다.
다만 css나 class에서 차이가 생겼다.
게다가 비효율적인 부분도 많았으니…
리액트로 한번 다시 만들어보았다.

 <!--[if lt IE 9]>

common.js

어느 순간부터 jQuery를 사용하지 않아도 문제가 없었기에 오랜만에 보니 감회가 새롭다.
옛날에는 스타일시트와 자바스크립트를 상단에 배치했었다.
그때는 위에 있든 아래에 있든 차이가 있을거란 생각조차 못했었다.
아무튼 항상 위에 두었기에 필연적으로 jQuery 사용시 아래와 같이 시작했다.

$(document).ready(function () {

}

	$("#work").find(".img_move>a").on({
		"mouseenter focus":function(){
			var imgW=$(this).width();
			var imgH=$(this).children().height();
			var marL=(imgW-(imgW*1.1))/2
			for (var i=0 ; i<11 ; i++ ) {
				$("#work").find(".img_move").eq(i).css({width:imgWH[i][0],marginLeft:0,height:imgWH[i][1]})
			}
			$(this).children().stop().animate({width:imgW*1.1,marginLeft:marL,height:imgH*1.1})
		},
		"mouseleave blur":function(){
			for (var i=0 ; i<11 ; i++ ) {
				$("#work").find(".img_move").eq(i).children("a").children().stop().animate({width:imgWH[i][0],marginLeft:0,height:imgWH[i][1]})
			}
		}
	});

IE 저버전에서도 보였으면 하는 마음에 css3는 거의 사용하지 않았었다.
아래와 같이 쉽게 구현가능하다.

  #work .img_move a{
    transition: transform 0.4s;
  }
  #work .img_move a:hover{
    transfrom: scale(1.1);
  }

transition을 이용하면 훨씬 부드러운 애니메이션을 볼 수 있다.
물론 가볍기도 하고…
