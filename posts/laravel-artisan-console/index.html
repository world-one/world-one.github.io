artisan 명령어 생성

php artisan make:command SendEmails

app/Console/Commands/SendEmails.php 파일이 생성된다. 

 protected $signature = 'email:send {user}';
 protected $description = 'send email';
  public function handle()
  {
      //
      echo "아직은 못보내요! ".$this->argument('user')."\n";
  }

내용을 간단히 입력하고 Kernel.php 파일에 추가해주었다.

  protected $commands = [
        //
        Commands\SendEmails::class
    ];

리스트에 위와 같이 명령어가 추가되어 있으면 기존 명령어와 같이 사용된다.

routes/console.php??

Class로 정의하는 위 방식 대신 클로저 기반 명령어를 만들 수 있는 곳이다.
Artisan::command를 통해 명령어를 쉽게 만들 수 있다.

Artisan::command('desc:console {desc}', function ($desc) {
    echo $desc."\n";
})->describe('text print');

$php artisan 
 desc
  desc:console         text print

$php artisan desc:console '클래스 말고 클로저 기반'
클래스 말고 클로저 기반

argument, option

protected $signature = 'email:send {user}';
protected $signature = 'email:send {user?}';
protected $signature = 'email:send {user=foo}';

필수로 받거나 선택적으로 받을 수 있으면 기본값도 설정할 수 있다. 

protected $signature = 'email:send {user} {--queue}';
protected $signature = 'email:send {user} {--queue=}';
protected $signature = 'email:send {user} {--queue=default}';

옵션값이 들어가면 true, 없으면 false가 된다.
옵션값을 지정해야하는 경우는 ’=‘을 붙인다.
기본값을 지정할 수 있다. 

email:send {user} {--Q|queue}

옵션의 단축키를 지정할 수 있다. 

email:send {user*}
email:send {user} {--id=*}
$php artisan email:send foo --id=1 --id=2

인자값과 옵션을 배열로 받을 수 있다. 

$userId = $this->argument('user');
$queueName = $this->option('queue');

값을 받는 방법이다. 배열일 경우 파라미터를 넣지 않으면 된다.

입력 설명

protected $signature = 'email:send
                        {user : The ID of the user}
                        {--queue= : Whether the job should be queued}';

다른 명령어 호출

 $this->call('email:send', [
    'user' => 1, '--queue' => 'default'
  ]);

  $this->callSilent('email:send', [
    'user' => 1, '--queue' => 'default'
  ]);

다른 명령어를 호출할 경우 사용한다. 출력무시가 필요하다면 callSilent를 사용하자.

프로그래밍 방식

Route::get('/foo', function () {
    $exitCode = Artisan::call('email:send', [
        'user' => 1, '--queue' => 'default'
    ]);
});
