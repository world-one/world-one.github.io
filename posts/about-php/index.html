내가 쓰고 있는 PHP에 대하여

입문

처음에는 신입 웹 퍼블리셔로 스타트업에 취업을 했다.
다만 경영진들은 내가 정확히 뭘하는 사람인지 몰랐나보다.
내가 모르는걸 잔뜩 물어보았고,
그럴때마다 ‘나는 잘 모르겠다, 좀 찾아보겠다’ 라는 말을 할 수 밖에 없었다.
그때 당시 외주를 받기로 했던 회사에서 추가 계약을 하지 않아서 같이 뽑혔던 디자이너들이
해고 당했고, 내 역할이 좀 애매해지던 시기였다.
개발팀장은 나에게 백엔드 작업을 해보자며 권유했고
당시 개발팀장의 주 언어인 JAVA와 함께 공부하며 작업해야하는 PHP, 두가지 선택지가 있었다.
아무래도 PHP가 좀 더 배우기 용이할 것이라며 PHP를 추천했고,
그렇게 백엔드 개발자로 시작하게 되었다.

PHP 뜻

처음에는 Personal Home Page Tools의 약자였으나
PHP3.0이 발표 될때 PHP: Hypertext Preprocessor로 변경되었다.

PHP 인식

구글링을 해보면 PHP에 대하여 부정적인 글들이 상당히 많이 나온다.
그런 글들을 보면 나도 불안감을 많이 느꼈다.
그럼에도 아직도 PHP를 쓰는 곳이 너무 많다. 생각보다 취업도 잘되는 것 같고…
버전업 될때마다 PHP의 성능도 엄청 좋아졌다고 한다.
php7에서 특히 그렇다고 하는데, 솔직히 그걸 느낄만큼 서비스가 지속된 적이 없다…
php8은 얼마나 좋아질지 기대가 된다.

Framework

애초에 PHP 작업을 Framework로 시작했다.
여러가지가 있지만 Codeigniter(CI)를 선택하여 API 서버를 만들어보았고
처음엔 뭘 잘모르고 MVC 패턴만 숙지하여 작업을 해나갔다. JWT토큰을 이용한 인증처리, GCM을 이용한 푸시메시지 전송 등 새로운 세상이었다.
CI는 국내 커뮤니티가 잘 조성되어 있다. 거기서 index.php 없애는 방법에 대한 글을 보고 문제를 해결한 기억이 있다.
가비아 독립몰을 구입해서 커스텀을 진행한 적이 있는데 고맙게도 CI로 제작되어 있어서
좀 더 수월하게 작업할 수 있었다.
그 이후로 이름이 가장 많이 거론되었던 Laravel을 사용해보았다. CI에 비하면 기본적으로 포함된게 많아서 좀 무거운 느낌이 들었지만
CI에서 쓰지 않는 네임스페이스를 사용한다던지( PSR 표준 ), Artisan 명령어나, 쿼리빌더는 작업을 참 편하게 할 수 있게 해준다.
아직도 못 써본 기능이 제법 많다.
거기에 blade나 vue.js까지, 프론트 작업에 대한 환경도 설정되어 있다.

PSR 표준

PHP-FIG(PHP-FrameWork Interop Group)는
유명한 PHP framework의 제작자들로 구성된 커뮤니티에서 만든 표준안이다.
총 20개중 현재 13개가 받아들여져있다.

앞으로

처음 개발팀장과 함께 PHP를 시작했을 때,
아무래도 JAVA로 작업을 해왔던 경험인지 엄청나게 빠르게 습득해서
마치 이전부터 계속 사용하던 언어마냥 만들어가는 걸 보며 감탄했었다.
지금와서 생각해보니, 단순히 JAVA개발자, PHP개발자같이 나눌 필요는 없는 것 같다.
개발자는 개발자고, 언어는 도구일 뿐.. 너무 갇혀있지 말아야한다.
기본에 충실하다면 어떤 언어든 금방 배우지 않을까 하는 생각이 든다.
